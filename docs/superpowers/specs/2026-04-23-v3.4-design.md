# EasyCord v3.4 — Events, Growth & DX

**Theme:** Every feature in this release helps bots grow their community and reduces the boilerplate tax developers pay just to set up a working bot.

**Goal:** Add three first-class capabilities — scheduled event management, invite tracking, and intent configuration shortcuts — making EasyCord demonstrably better than raw discord.py for all three.

**Architecture:** Scheduled events and invites extend the `Context` object via the existing `_context_channels.py` mixin. Invite tracking ships as a new `InviteTrackerPlugin` that uses `bot.db` and `SlashGroup`. Intents shortcuts are pure Composer additions. No new files except the plugin and its tests.

**Tech Stack:** discord.py 2.x, existing `EasyCordDatabase` (SQLite/memory), existing `SlashGroup` pattern, existing `_context_channels.py` mixin.

## Feature 1: Scheduled Events

### Why this is better than discord.py

discord.py: `await guild.create_scheduled_event(name=..., start_time=..., entity_type=..., privacy_level=..., channel_id=...)` — callers must construct enums, pass channel IDs manually, and remember `end_time` is required for external events.

EasyCord: `await ctx.create_event("Movie Night", start)` — sensible defaults, auto-detects event type from what you pass, raises `RuntimeError` with a human message when `end_time` is missing for external events.

### API

Add to `easycord/_context_channels.py`:

```python
async def create_event(
    self,
    name: str,
    start_time: datetime,
    *,
    end_time: datetime | None = None,
    description: str | None = None,
    channel: discord.abc.GuildChannel | None = None,
    location: str | None = None,
    image: bytes | None = None,
) -> discord.ScheduledEvent:
    """Create a scheduled event in the guild.

    Pass ``channel`` for a voice/stage event; pass ``location`` (a string)
    for an external event. External events require ``end_time``.
    Omit both for a generic guild event.
    """

async def get_events(self) -> list[discord.ScheduledEvent]:
    """Return all upcoming scheduled events for this guild."""

async def delete_event(self, event: discord.ScheduledEvent | int) -> None:
    """Cancel and delete a scheduled event by object or ID."""
```

### Implementation notes

- `create_event` must guard against `location` without `end_time` with `RuntimeError("External events require end_time")`.
- `create_event` must guard against `channel` and `location` both provided with `RuntimeError`.
- Entity type is inferred: `channel` → `EntityType.voice` or `EntityType.stage_instance` (check `isinstance(channel, discord.StageChannel)`); `location` → `EntityType.external`; neither → `EntityType.external` with `location=""` is not valid, raise `RuntimeError("Provide channel or location")`.
- `delete_event` accepts either a `discord.ScheduledEvent` object or an int ID; fetch via `guild.fetch_scheduled_event(id)` if an int is passed.
- Privacy level is always `PrivacyLevel.guild_only` (the only supported value in Discord's API).
- Requires `guild_scheduled_events` intent — document this in docstring.
- All three methods raise `RuntimeError("create_event requires a guild context")` if `self.guild is None`.

### Required intent

Callers must enable `guild_scheduled_events`. The `Composer` intents shortcuts (Feature 3) include `.with_events()` to make this easy.

## Feature 2: Invite Management

### Why this is better than discord.py

discord.py: `await channel.create_invite(max_uses=10)` — works, but there's no built-in way to track who invited whom. Building invite attribution requires ~30 lines of boilerplate every developer writes from scratch.

EasyCord: `ctx.create_invite()` is the quick API. `InviteTrackerPlugin` handles the boilerplate automatically — just add the plugin and query `/invite stats`.

### 2a. Context methods

Add to `easycord/_context_channels.py`:

```python
async def create_invite(
    self,
    *,
    max_uses: int = 0,
    max_age: int = 0,
    temporary: bool = False,
    unique: bool = True,
    reason: str | None = None,
) -> discord.Invite:
    """Create an invite for the current channel.

    ``max_uses=0`` means unlimited; ``max_age=0`` means never expires.
    Returns the created ``discord.Invite`` object.
    Raises ``RuntimeError`` if called outside a guild or text channel.
    """

async def get_invites(self) -> list[discord.Invite]:
    """Return all active invites for the current channel."""

async def revoke_invite(self, code: str) -> None:
    """Delete an invite by its code."""
```

### 2b. InviteTrackerPlugin

New file: `easycord/plugins/invite_tracker.py`

**What it does:**
1. Snapshots all guild invites on `on_ready` and `on_guild_join`.
2. On `on_member_join`, fetches current invites, diffs against snapshot to identify which code gained a use, records the attribution, updates the snapshot.
3. Stores attribution in `bot.db` under key `invite_tracker:{guild_id}:{invite_code}` as JSON: `{"inviter_id": int, "uses": int, "code": str}`.
4. Exposes `/invite stats [@user]` and `/invite top [limit=10]` slash commands.

**Race condition note:** If two members join within the same second, the snapshot diff may misattribute one. This is unavoidable with the Discord API; document it in the class docstring.

```python
class InviteTrackerPlugin(SlashGroup, name="invite"):
    """Track which invites members used when joining the server."""

    def __init__(self) -> None:
        super().__init__()
        self._snapshots: dict[int, dict[str, int]] = {}  # guild_id → {code: uses}
```

**Required intents:** `members` (for `on_member_join`), `guild_invites` (for `guild.invites()`). Document in class docstring. The `.with_members()` and `.with_invites()` shortcuts (Feature 3) enable these.

**Slash commands:**

`/invite stats [@user]` — shows how many members a user has invited. If no user given, shows the invoker's stats. Ephemeral.

`/invite top [limit=10]` — shows the top `limit` inviters by total uses. Embed response.

**Database schema (key-value):**
- Key: `invite_tracker:{guild_id}:{code}` → `{"code": str, "inviter_id": int | None, "uses": int}`
- Key: `invite_tracker:snapshot:{guild_id}` → `{"code": int, ...}` (uses count per code)

**Plugin is added via `add_group()`, not `add_plugin()`** — same pattern as `ReactionRolesPlugin`. It cannot go in `builtin_plugins.py`.

## Feature 3: Intents Shortcuts on Composer

### Why this is better than discord.py

discord.py + raw bot:
```python
intents = discord.Intents.default()
intents.members = True
intents.message_content = True
client = discord.Client(intents=intents)
```

EasyCord with shortcuts:
```python
bot = Composer().with_members().with_messages().log().build()
```

### API

Add to `easycord/composer.py`:

```python
def with_members(self) -> Composer:
    """Enable the privileged Members intent (required for member join/leave events)."""

def with_messages(self) -> Composer:
    """Enable the privileged Message Content intent (required to read message text)."""

def with_reactions(self) -> Composer:
    """Enable the Reactions intent (required for reaction add/remove events)."""

def with_presence(self) -> Composer:
    """Enable the privileged Presences intent (required for online status tracking)."""

def with_voice(self) -> Composer:
    """Enable the Voice States intent (required for voice channel join/leave events)."""

def with_events(self) -> Composer:
    """Enable the Guild Scheduled Events intent."""

def with_invites(self) -> Composer:
    """Enable the Guild Invites intent (required for invite tracking)."""

def with_all(self) -> Composer:
    """Enable all intents (equivalent to discord.Intents.all())."""
```

### Implementation notes

- Each `.with_X()` method calls `self._ensure_intents()` which returns `self._intents` if set, otherwise initializes to `discord.Intents.default()`.
- Calling `.with_all()` replaces `self._intents` entirely with `discord.Intents.all()` — it does not merge.
- Calling `.intents(custom_intents)` after `.with_members()` replaces the accumulated intents — document this in `.intents()` docstring.
- The shortcuts do **not** validate whether the intent requires portal approval; that's Discord's job at connection time.

### Composing shortcuts with existing `.intents()`

The existing `.intents()` method stays unchanged. Shortcuts and `.intents()` are alternatives — developers use one or the other, not both. Document this clearly: "Use `.with_*()` shortcuts for convenience or `.intents()` for full control; don't mix them."

## Testing

All three features follow TDD (write failing test, implement, verify passing).

**Scheduled events:** Mock `guild.create_scheduled_event`, `guild.fetch_scheduled_events`, `guild.fetch_scheduled_event`, `guild.delete_scheduled_event`. Verify correct `entity_type` inference, `RuntimeError` on missing `end_time` for external events, `RuntimeError` outside guild.

**Invite management (context):** Mock `channel.create_invite`, `channel.invites`, `discord.utils.get`. Verify `RuntimeError` outside guild.

**InviteTrackerPlugin:** Use `MemoryDatabase`. Mock guild invite lists. Verify snapshot logic, attribution recording, and correct diff when one code gains a use. Test `/invite stats` and `/invite top` command handlers.

**Composer shortcuts:** Unit test — construct `Composer`, call shortcuts, call `.build()`, assert `bot._intents` has the expected flags set.
